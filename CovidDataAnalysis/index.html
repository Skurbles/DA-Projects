About This Project

Covid-19 has impacted and transformed the world as we know it completely. Since the beginning there was uncertainty as to the scope and outcome of this worldwide pandemic. Fortunately, through the use of data science we can now see with accuracy the true nature of the virus. Use of data science in such a pandemic will lead to greater insights in the data we are working on and assist in the treatment and understanding of the threat. This project is a demonstration of the type of insights data can provide while showing how different types of SQL queries and visualization using Tableau can assist in making that more easily digestible.

Data Cleaning and Preparation

The first step to analysing data is getting it prepared for use. In the case of this dataset certain values had to be added, and others transformed. Using Excel I was able to add necessary missing values to NULL and transform necessary values to "0" for use with the MySQL database used. Following this, corresponding tables needed to be created in the database to match the original dataset columns. This cleaning process properly prepared the dataset for import using a LOAD DATA INFILE command. For this project, 2 tables were created. One focusing on the covid death data, and another focusing on the covid vaccines data.

Querying the Database with SQL

Once the initial database is created and we have verified that it is complete and correct, we can begin the queries in order to return the specific data that we need for analysis. In this image are some examples of the queries used to generate the visualization shown later in this project. As you can see in my initial queries we start by querying the database for some simple totals as well as some percentage of death by country results. Looking at the total cases vs population we can see an example query that returned a result showing that roughly 30% of the US population has contracted Covid-19.

Queries Continued

In this next example you can see that I was querying the db for looking at highest deathcount per population Ordered by country. From there we can start breaking things down worldwide by continent. Following that I was able to organize the results by date and then total time over the course of the outbreak. Lastly in this example we can see that I was able to do a simple join to get data simultaneously from the covid_deaths table and the covid_vaccines table.

Queries and CTE

Here you can see an example of a slightly more complex join query pulling data from both tables. This join aligns the data on the common fields to show vaccines by country displayed by location country and then date recorded. The RollingPeaopleVaccinated field designated here represents a column that is created and added that shows a running total by country by date on the outside of the table from the beginning of Covid up until the creation of this project.

CTE & Temp Table

In this last example you can see that I was demonstrating the ability to create a Temp Table. This was to reference the previous CTE and give us a table that will not need to be saved forever and allows an easier reference for a specific data result for easier reference in subsequent queries.

Results and Visualizations

Up to this point we have looked at the queries used for getting results in the database. How can we show the results in a meaningful way? There are many ways to make the query results more easily understandable, but for this project I used Tableau to show some examples of visualizing the results. Below are some examples of the individual queries that help turn data into a story.

Interactive World Map

Ineractive World Maps can show tons of data using just color. For example in this image, the saturation of color represents the percentage of the population that has contracted Covid displayed worldwide by country. For example you can see here that when the US is hovered over it shows that 30% of the population has contracted Covid.

Interactive Line Graph

Interactive line graphs are great at showing progression of a value over time. There are many ways to isolate and display the data as well whether it is a simple single value graph, a multiple source graph like the one we have here or other more basic graphs. On the graph we have here you can see that when you hover over the displayed point you can see that on April of 2021 the US had 32,468,043 total infections at that time.

Packed Bubble Graph

Packed bubble graphs are another great way of displaying a value in many ways all at once. In the example here we can see that the size of the bubbles represent the relative size of the population while the color saturation represents the relative number of vaccinations. As you can see in the example here Vietnam has a relatively small population, but a large vaccinated percentage. Unlike India where they have a massive population, but relatively few vaccinations.

Interactive Bar Chart

Interactive bar charts are just one more great way of displaying many values. Bar charts are great for a multitude of data types and are quite versatile. In the example here we see the inhabited continents listed on the x axis and the amount of total deaths attributed to Covid on the vertical y axis.

Interactive Dashboards

Interactive dashboards are some of the best tools at the disposal of an analyst. They are the perfect way to compile a multitude of data and queries from any number of sources in one place for easy consumption, presentation, or sharing. It is a way to make the story of numbers, values, and variables spring to life and tell a story. Sometimes it is a simple one meant to quickly provide requested informaiton. Sometimes it is a way to show opportunities in business directions, operations, or sales. Whether it is a simpler one like the one displayed here, or a very complex multimedia report with text, graphics, and interactive charts, a dashboard is the perfect way to lay out the story for your intended audience.

By utilizing dashboards users are able to filter the data that they need and discover the correct insights in an easy way. The filters can be applied without the help of anyone else and consult the information as they need it. By having this information available right away correct decisions can be made. On average it can take anywhere from a few weeks to a few months or more to generate a dashboard for a project. The development is dependent on a couple of processes that need to be completed as well as the scope of the project itself. The elements that also contribute to the development of a dashboard are the selection of KPI's (Key Points of Interest), dataset cleaning/preparation, and dashboard design and development.
